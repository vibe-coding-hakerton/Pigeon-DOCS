# Pigeon 테스트 시나리오

> **작성일**: 2025-12-11
> **버전**: v1.0
> **상태**: Draft

---

## 1. 개요

이 문서는 Pigeon 서비스의 주요 기능에 대한 테스트 시나리오를 정의합니다.

### 1.1 테스트 범위

| 구분 | 영역 | 우선순위 |
|------|------|----------|
| E2E | 사용자 시나리오 기반 전체 흐름 | P0 |
| Integration | API 및 서비스 간 연동 | P0 |
| Unit | 개별 컴포넌트 및 함수 | P1 |

### 1.2 테스트 도구

- **E2E**: Playwright
- **Frontend Unit**: Vitest + React Testing Library
- **Backend Unit**: pytest + pytest-django

## 2. 인증 (Authentication)

### TC-AUTH-001: Gmail OAuth2 로그인 성공

**우선순위**: P0

**전제조건**:
- 사용자가 유효한 Gmail 계정을 보유
- Google Cloud Console에 OAuth2 설정 완료

**테스트 단계**:
1. 랜딩 페이지(`/`)에 접속
2. "Gmail 연동하기" 버튼 클릭
3. Google OAuth 동의 화면으로 리다이렉트 확인
4. Google 계정 로그인 및 권한 승인
5. Pigeon 콜백 URL로 리다이렉트
6. 메인 화면(`/mail`)으로 이동

**예상 결과**:
- [ ] JWT Access Token 발급
- [ ] JWT Refresh Token 발급
- [ ] 사용자 정보 저장 (email, name, picture)
- [ ] 메인 화면 정상 표시

## 11. 테스트 데이터

### 11.1 테스트 계정

테스트 계정 정보는 환경 변수로 관리합니다. 자세한 설정은 `.env.backend.example` 파일을 참조하세요.

```bash
# 백엔드 .env 파일에 설정
TEST_GMAIL_ACCOUNT_1=your-test-account-1@gmail.com
TEST_GMAIL_ACCOUNT_2=your-test-account-2@gmail.com
```

| 환경 변수 | 용도 |
|-----------|------|
| `TEST_GMAIL_ACCOUNT_1` | 일반 테스트 |
| `TEST_GMAIL_ACCOUNT_2` | 멀티 유저 테스트 |

> **주의**: 테스트 계정은 반드시 테스트 전용 Gmail 계정을 사용하세요. 실제 업무용 계정을 사용하지 마세요.

### 11.2 테스트 메일 시나리오

| 시나리오 | 메일 수 | 설명 |
|----------|---------|------|
| 업무 메일 | 20개 | 프로젝트 관련, 회의록 등 |
| 알림 메일 | 15개 | GitHub, Jira, Slack |
| 뉴스레터 | 10개 | 기술 뉴스, 마케팅 |
| 쇼핑/개인 | 10개 | 배송 알림, 영수증 |
